### 2. 主 session 只调度，执行全 spawn

> 来源：kloss_xyz 帖子里的 Advanced Principles

**核心原则：**
- 主 session 只负责策略和调度
- 所有执行都 spawn 子 agent
- 看到错误立刻修复不等待
- 外部输出（邮件/推文/API 写入）一律先安全门控确认再执行

这和 OpenClaw 最佳实践高度吻合，值得固化进自己的 SOUL.md。

### 3. Secrets 迁移最佳实践

> 来源：官方配套 | 2026.2.26

steipete 原话：
> "Your API keys don't belong in plaintext config files."

**三种 Provider：**
- ENV（读系统环境变量）
- File（指定路径文件）
- Exec（运行命令动态获取，推荐接 AWS SSM 等）

**迁移流程：**
```
openclaw secrets audit → configure → apply → reload
```
